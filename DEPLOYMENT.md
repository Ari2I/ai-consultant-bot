# Развёртывание на сервере

## Сертификат для GigaChat

GigaChat API использует TLS-сертификат, подписанный российским
удостоверяющим центром (НУЦ Минцифры), которого нет в стандартном
доверенном списке корневых сертификатов ни в одной ОС/дистрибутиве
Docker-образа по умолчанию. Без этого сертификата запросы к GigaChat
будут падать с ошибкой проверки TLS.

**Что сделать:**

1. Скачайте сертификат с официального источника:
   https://www.gosuslugi.ru/crt (раздел «Сертификат НУЦ Минцифры»).
2. Положите файл в `./certs/` в корне проекта (см.
   `certs/README.md`).
3. В `.env` укажите путь к файлу:
   - для запуска в Docker: `GIGACHAT_CA_BUNDLE_FILE=/app/certs/имя_файла.cer`
     (каталог `./certs` примонтирован в контейнер — см.
     `docker-compose.yml`);
   - для запуска без Docker: укажите реальный путь на сервере,
     например `GIGACHAT_CA_BUNDLE_FILE=/home/botadmin/ai_consultant_bot/certs/имя_файла.cer`.

Сам файл сертификата не входит в репозиторий (исключён в
`.gitignore`/`.dockerignore`) — скачивайте его самостоятельно с
официального источника при каждом развёртывании.

**Быстрая проверка без сертификата (только для локальной отладки):**
`GIGACHAT_VERIFY_SSL_CERTS=false` в `.env` отключает проверку TLS
целиком. Подходит, чтобы быстро убедиться, что бот вообще может
достучаться до GigaChat, но **не годится для продакшена** — на
реальном сервере обязательно настройте сертификат правильно.

## Особенность этого проекта: доступ к HuggingFace Hub при первом запуске

При **первом** запуске бота библиотека sentence-transformers
скачивает файлы модели эмбеддингов
(`paraphrase-multilingual-MiniLM-L12-v2`, около 470 МБ) с
`huggingface.co`. Это значит:

- Серверу нужен исходящий доступ в интернет к `huggingface.co` (и
  связанным CDN) хотя бы один раз — при первом старте контейнера.
- После первого успешного запуска модель кэшируется локально (см.
  `docker-compose.yml` — volume `hf_cache`), и повторные перезапуски
  контейнера сеть для этого уже не используют.
- Если сервер находится в закрытом контуре без доступа к
  huggingface.co вообще — модель нужно будет скачать заранее на
  машине с доступом и скопировать кэш вручную, либо использовать
  прокси для исходящего трафика к huggingface.co (отдельная задача,
  не входящая в этот гайд).

Ни к прокси для Telegram (см. ниже), ни к сертификату для GigaChat
(см. выше) это не имеет отношения — три независимых внешних сервиса
с независимыми требованиями к сети/сертификатам.

## 1. Выбор и первоначальная настройка сервера

Подойдёт недорогой VPS у любого провайдера — конкретных цен не
называю, так как не могу проверить их актуальность прямо сейчас,
сверьтесь на сайте перед покупкой. Учитывая, что sentence-
transformers тянет за собой PyTorch, требования выше, чем у прошлых
двух проектов: минимум 2 vCPU и 2 ГБ RAM (в `docker-compose.yml`
установлен лимit `mem_limit: 1g` — при необходимости увеличьте под
реальную нагрузку).

После получения доступа по SSH:

```bash
sudo apt update && sudo apt upgrade -y
sudo adduser botadmin
sudo usermod -aG sudo botadmin
```

## 2. Базовая защита сервера

Аналогично прошлым проектам:

```bash
# На своей машине, если ключа ещё нет:
ssh-keygen -t ed25519
ssh-copy-id botadmin@ваш_сервер
```

В `/etc/ssh/sshd_config`:

```
PasswordAuthentication no
PermitRootLogin no
```

```bash
sudo systemctl restart sshd

sudo apt install ufw -y
sudo ufw allow OpenSSH
sudo ufw enable

sudo apt install unattended-upgrades -y
sudo dpkg-reconfigure --priority=low unattended-upgrades
```

## 3. Установка Docker

```bash
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker $USER
newgrp docker

docker --version
docker compose version
```

## 4. Загрузка проекта и настройка

```bash
scp -r ./ai_consultant_bot botadmin@ваш_сервер:~/
cd ~/ai_consultant_bot
cp .env.example .env
nano .env
```

Обязательно заполните в `.env`:

- `BOT_TOKEN` — токен от [@BotFather](https://t.me/BotFather).
- `ADMIN_CHAT_IDS` — chat_id владельца бизнеса (и менеджеров, если
  есть), через запятую. Узнать свой chat_id можно, написав боту
  [@userinfobot](https://t.me/userinfobot).
- `GIGACHAT_CREDENTIALS` — Authorization key из личного кабинета
  https://developers.sber.ru/studio/.
- `GIGACHAT_CA_BUNDLE_FILE` — путь к сертификату (см. раздел выше).

Остальные параметры (порог уверенности, размер чанков, лимит
запросов в минуту) можно оставить по умолчанию и подобрать позже по
факту использования — см. README.

Создаём каталог для базы данных и кладём сертификат:

```bash
mkdir -p data certs
chmod 777 data
# скопируйте сюда файл сертификата, скачанный с gosuslugi.ru:
# scp russian_trusted_root_ca.cer botadmin@ваш_сервер:~/ai_consultant_bot/certs/
```

## 5. Запуск

```bash
docker compose up -d --build
```

Первый запуск займёт заметно больше времени, чем обычно — контейнер
качает зависимости (включая PyTorch) при сборке образа, а затем сам
бот при первом старте скачивает модель эмбеддингов, как описано выше.

Проверка логов:

```bash
docker compose logs -f
```

В логе должна появиться строка о запуске polling без ошибок. Если
видите ошибку про `GIGACHAT_CREDENTIALS`, `BOT_TOKEN` или проверку
TLS-сертификата — проверьте `.env` и наличие файла в `./certs`.

Остановка / перезапуск:

```bash
docker compose down
docker compose up -d
```

## 6. Автозапуск после перезагрузки сервера

```bash
sudo systemctl enable docker
```

`restart: unless-stopped` в `docker-compose.yml` поднимет контейнер
сам после сбоя или перезагрузки сервера.

## Загрузка демо-документов (для теста после развёртывания)

В комплекте есть готовые демо-документы вымышленного магазина
одежды «ТриНитки» (`demo_data/`). Чтобы быстро проверить, что бот
работает:

1. Напишите боту от имени админа команду `/kb_add`.
2. Отправьте по очереди `demo_data/about.txt`, `demo_data/faq.docx`,
   `demo_data/price_list.xlsx`, `demo_data/shipping_returns.pdf` —
   для каждого файла заново вызывайте `/kb_add`.
3. Проверьте `/kb_list` — должно быть 4 документа.
4. Напишите боту не от админа (с другого аккаунта) вопрос вроде
   «Какой у вас режим работы?» или «Сколько стоит доставка по
   Москве?» — бот должен ответить на основе загруженных документов.
5. Задайте заведомо нерелевантный вопрос («Делаете ли вы
   татуировки?») — бот должен эскалировать его админам, а не
   выдумывать ответ.

## Права на каталог данных

Как и в прошлом проекте — пользователь внутри контейнера (`appuser`)
не root, поэтому `chmod 777 data` перед первым запуском остаётся
самым простым решением для одного контейнера с единственным
назначением.

## Резервное копирование

База знаний и журнал обращений клиентов хранятся в одном файле
SQLite (`./data/knowledge_base.db`):

```bash
# Пример cron-задачи (crontab -e), копирует базу раз в сутки в 3:00
0 3 * * * cp ~/ai_consultant_bot/data/knowledge_base.db ~/backups/knowledge_base_$(date +\%F).db
```

## Что не входит в этот гайд

- Настройка HTTPS/домена — не нужна, бот использует long polling, а
  не webhook.
- Настройка обхода блокировок Telegram Bot API (если она актуальна в
  вашем регионе) — поддержка уже встроена через `PROXY_URL` в
  `.env` (см. README, раздел «Блокировка Telegram Bot API в
  России»), нужно только подставить рабочий прокси-сервер.
- Настройка проксирования доступа к huggingface.co для закрытых
  корпоративных сетей — см. пояснение выше.
- Мониторинг падений и алерты — можно добавить отдельно (Uptime
  Kuma и аналоги), если станет реальным требованием заказчика.
- Миграция с SQLite на PostgreSQL при росте нагрузки — код уже
  поддерживает смену `DATABASE_URL`, сама миграция в объём этого
  гайда не входит.
